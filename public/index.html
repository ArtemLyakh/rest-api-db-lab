В ссылках *** определяет базу данных. Для доступа к NoSQL базе нужно подставлять в путь mongo, для доступа к SQL - mysql.

Общие сведения

При выполнении методов возможны 2 варианта: успешное выполнение, ошибка.
В случае успешного выполнения возвращается код 200 и рабочий объект. Либо возвращается объект вида: 

        {
            success: true
        }
для методов, которые не могут ничего возвращать (например, удаление).

В случае ошибки возвращается код ошибки (400, 403, 404) описанием ошибки в следущем виде: 

        {
            error: "Описание возникшей ошибки"
        }

В случае каких-либо ошибок на сервере возвращается код 5хх с описанием.

POST /reset

Метод полностью сбрасывает указанную базу данных и заполняет ее тестовыми значениями (2 платформы, 3 игры, 2 пользователя) со связями.
Не принимает входных параметров.
По окончанию выполнения возвращается код 200.

Назад

GET /platforms

Список всех платформ.
Возвращается массив объектов следующего вида: 

        {
            "_id": #ID#,
            "name": "name",
            "company": "company",
            "controller": "controller",
            "store": "store"
        }

Фильтрация

При запросе возможна фильтрация данных по полям name, company, controller, store.
При фильтрации производится регистронезависимое сопоставление по подстроке.
Следующий запрос выберет элементы с именем, где встречается "qwe" и компанией "test": 

        get /***/platforms?name=qwe&company=test

Сортировка

При запросе возможна сортирвка по полям name, company, controller, store.
За сортировку отвечают параметры параметры sort и order.
Параметр sort принимает имя поля по которому должна производится сортировка.
Параметр order отвечает за направление сортировки. Имеет сысл только при наличии парамтра sort. Допустимые значения: asc, desc. Если не указан параметр order, то сортировка будет произведена для asc.
Если передано недопустимое значение, то возвращается код 400 с описанием ошибки. Следующий запрос выберет элементы, отсортированные по полю name по убыванию: 

        get /***/platforms?sort=name&orser=desc

Пагинация

По умолчанию выборка ограничена 10ю элементами.
Для изменения размера выборки и управления пагинацией используются параметры limit и skip.
Следующий запрос выберет 50 элементов, пропустив 200 элементов: 

        GET /***/platforms?limit=50&skip=200

Параметр limit должен быть целочисленным и лежать в интервале [0,100], skip должен быть положительным целым числом.
В случае неверных значений параметров возвращается код 400 с описанием ошибки.

GET /games

Список всех игр.
Возвращается массив объектов следующего вида: 

        {
            "_id": #ID#,
            "name": "name",
            "genre": "genre",
            "rating": 100,
            "releases": [
                ...
            ]
        }

Где поле releases является массивом объектов, соответствующих релизам данной игры на конретной платформе: 

        {
            "platform": { ... },
            "price": 100,
            "date": 1493319189
        }

Где поле date представляет собой время в формате Unix timestamp, поле platform является объектом следующего вида: 

        {
            "_id": #ID#,
            "name": "name"
        }

Поле _id соотвествует id конкретной платформы, поле name - ее названию.

Фильтрация

При запросе возможна фильтрация данных по полям name, genre, rating.
При фильтрации для поле name, genre производится регистронезависимое сопоставление по подстроке.
Для фильтрации по рейтингу необходимо передать параметр rating в формате: 

        rating=100
или 
        rating=50;150
В первом случае будет произведена фильтрация игр у которых рейтинг равен 100. Во втором случае вернутся игры у которых рейтинг лежит в интервале [50,150].
Если необходимо задать только 1 из границ интервала, то необходимо указывать параметр следующим образом: 
        rating=;150
или 
        rating=50;
В первом случае вернутся все игры, у которых рейтинг меньше или равен 150. Во втором случае вернутся все игры, у которых рейтинг больше или равен 50.

PUT /users/#ID#/libraries/#ID#/games

Добавление новой игры в указанную библиотеку конкретного пользователя.
В качестве входного параметра принимется объект следующего вида: 

            {
                "game": #ID#
            }

Параметр game соответствует id конкретной игры.
Параметр game является обязательным. При его отсутствии вернется код 400 с соответствующей ошибкой.
При перечаче неверного формата параметра platfrom вернется код 400 с соответствующей ошибкой.
Добавить игру в библиотеку возможно только при условии того, что эта игра вышла для платформы указанной библиотеки. Если для платформы текущей библиотеки игры не вышла вернется код 403 с соответствуюшей ошибкой.
Также в библиотеке возмаожна только одна копия игры. При попытке добавить еще одну вернется код 403 с соответствуюшей ошибкой.
В случае успеха вернется код 200 с созданным объектом.
